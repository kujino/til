## 初めての本番DB接続
### 発生した問題
ローカルでカラムを追加したりしたために本番環境との相違が出来てしまってデプロイがうまくいかなかった

具体的には、ローカルのpostsテーブルにあらたにカラムを追加したため本番DBのデータには無い値が出来てしまったことが原因

#### 解決した方法

1. 以下を実行しpstglのクライアントルールをWSLにインストール

```ruby
sudo apt update
sudo apt install postgresql-client
```

2. その後Renderにて使っているDBの`**PSQL Command**`をコピーして実行  
3. 無事本番DBにアクセス出来たのでSQLを発行  

```ruby
SELECT * FROM posts;
```

4. 以前のテスト投稿が残っていたので削除

```ruby
DELETE FROM posts WHERE id = 1;
```

5. 最後に無事消えたか確認して再デプロイ
6. 成功！

初めてのデプロイエラーでビビった。
